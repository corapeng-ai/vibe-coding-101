Where to Host the Repo

Internal Team → GitLab

Your company's existing Git. Access controls, CI/CD, code review. Standard for enterprise teams.

Public / Community → GitHub

Conference attendees, open-source. GitHub is the default for discoverability and community sharing.

Both

Internal GitLab for proprietary workflows. Public GitHub for conference materials and templates others can fork.

Key Insight

Skills are markdown. Workflows are JavaScript. They travel with your repo. Anyone who clones it can run it. No deployment step.
